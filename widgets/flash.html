Introduction

The purpose of this document is to define consumption of the RateCity (http://www.ratecity.com.au) Flash Widget. You are assumed to be familiar with HTML and JavaScript programming, along with CSS (Cascading Style Sheets) to control the style.

Contractual Agreement

These widgets are provided to our partners to integrate RateCity into partner sites. If you would like to integrate these widgets onto your site, please email us: alex (dot) parsons (at) ratecity (dot) com (au) au. Any revenue will not be paid to widget users without agreements.

Implementation Details

The flash binary should be trafficked by your preferred ad-server. It’s pre-configured to pass back the page it runs on when requesting data feeds. This allows ratecity to configure the feed to ensure clicks are correctly attributed to you.

We provide two extra parameters which can be useful if you wish to run the flash binary across many sites and/or segment your clicks by placement. The parameter names are analogous to the Urchin/GA implementation for ease of explaining the concepts.

utm_source
This is the partner variable which was originally supplied by RateCity
utm_campaign
The variable allows click segmentation by whatever value you choose to put in here.
